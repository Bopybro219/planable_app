Developer preview

Planira API

An early JSON lookup preview for teams who want trusted place signals in their own workflows.

Authentication

Use a Bearer API key

Send your API key in the `Authorization` header. Raw keys are shown only once when they are created, so keep them private and rotate them if you no longer trust where they have been stored.

Header Authorization: Bearer {{ example_api_key }}
Endpoint GET /api/v1/places/search?q=pub&town=Northampton
Lookup rules

Calm limits for an early preview

Who can create keys Only paid, business, or staff access can create API keys. Free search access does not unlock the API.
Required query Send at least one of `q`, `town`, or `postcode`.
Lookup credits API lookup credits stay separate from member search credits.
Early preview note API packs are still a preview. During local and early testing, lookup credits may be assigned manually per key.
Access responsibility Keep keys with trusted people only. Planira may revoke keys that are abused, shared carelessly, or used against staff-only workflows.
Example request

Try a place search

curl -H "Authorization: Bearer {{ example_api_key }}" "{{ api_search_url }}?q=pub&town=Northampton"
